# Well Level Bridge v0.0.1

Initial release.

This release adds a HACS-compatible Home Assistant integration for migrating well-level retrieval out of Node-RED.

Highlights:

- Native Home Assistant sensors:
  - `sensor.well_level_bridge_raw`
  - `sensor.well_level_bridge_filtered`
- TCP serial bridge support with defaults for `192.168.10.4:9001`.
- Configurable parser settings:
  - frame delimiter
  - primary token index
  - fallback token index
  - sign inversion
- Configurable filtering:
  - moving-average window
  - filtered update interval
  - accepted minimum and maximum values
  - maximum allowed filtered delta
- Safe cutover behavior:
  - **Connect on start** defaults to off because the serial-to-Ethernet bridge accepts only one TCP connection.

The integration intentionally uses bridge-specific entity names by default. After cutover and verification, the filtered entity can be renamed to `sensor.well_level` in Home Assistant if you want future history to continue under the existing production entity ID.
